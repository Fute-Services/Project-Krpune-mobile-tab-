NETWORK ARCHITECTURE DIAGRAM

Project / Product Name: KRPUNE Native (krpune-native) - offline real estate showcase app
Prepared By: Fute Services (derived from source code review)

Version: 1.0.0

1. PURPOSE
Document the network footprint of the KRPUNE Native mobile app, based on inspection of its source code.

2. SCOPE
Client-side mobile app only (Android/iOS). This app has no backend server of its own and makes no network calls during normal use.

3. COMPONENTS FOUND IN CODE
- [x] End users / client devices (Android/iOS phones and tablets running the Expo app)
- [ ] Load balancers - none, no backend
- [ ] Web servers / application servers - none, app is fully offline/bundled
- [ ] Databases - none, all content is bundled as local JSON files (src/offline/data/*.json)
- [ ] Firewalls - not applicable, no server infrastructure to protect
- [ ] VPN / remote access points - none
- [ ] Cloud services used - none contacted at runtime; see note below on Cloudinary
- [ ] On-premise servers - none
- [ ] Third-party integrations / APIs - none
- [ ] Internal network segments / VLANs - not applicable, single client app

Note on Cloudinary: a handful of files (src/screens/Home.tsx, Amenities.tsx, Location.tsx) contain
strings that look like Cloudinary URLs. On closer inspection these are only used as lookup keys
passed into resolveAsset() (src/offline/resolveAsset.ts), which matches the filename against a
locally bundled asset map (src/offline/assetMap.ts) and returns the on-device file. All four of
these specific assets were confirmed present in the local bundle, so no network request is ever
made for them in normal use. resolveAsset() does have a defensive fallback that would stream from
the URL directly if an asset were ever missing from the bundle, but that path is not exercised
today - it exists only so the app does not hard-crash if a future asset is left out of the bundle
by mistake.

4. NETWORK ZONES
There is no server-side infrastructure and no live network activity for this app. It is a fully self-contained, offline mobile application - every screen, image, video, and data file ships inside the installed app.
