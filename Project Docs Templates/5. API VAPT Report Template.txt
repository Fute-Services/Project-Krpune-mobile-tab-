NOT APPLICABLE - NO API IN THIS PROJECT

A full source code review of the krpune-native repository (see "4. Mobile App Source Code Review Report") confirmed the app has no backend API of its own. All content (floor plans, amenities, mobility, transport, gallery, VR tour data) is bundled into the app at build time and served locally via src/api/localClient.ts, which only reads from local JSON files - it does not make any network calls. The only outbound network activity found in the code is fetching static images/video from the Cloudinary CDN over HTTPS, which is a public media delivery service, not a project API.

Because there is no API to test, this document is submitted as N/A for the IT team's review checklist rather than left blank. If a backend API is introduced later (for example, if the app moves from bundled offline data to a live content-managed backend), an API VAPT should be commissioned at that point using a qualified pentest vendor.
